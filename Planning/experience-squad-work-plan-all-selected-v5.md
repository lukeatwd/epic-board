## **Epic 列表（体验提升小分队的“主题战场”）**

- **E1 可达性与稳定性底座（网络/性能）**：聚焦网络可达性解释与性能基线，减少“能打开但用不了”的挫败。  
- **E2 身份与登录（注册/登录/验证码/账号管理）**：身份链路畅通是同步、计划、付费等的前置。  
- **E3 同步与数据资产安全（进度/恢复/冲突）**：用户资产不可丢；同步可见、可恢复、可解释。  
- **E4 找经文与搜索（精确/繁简/原文/语义/全局）**：查经效率与专业用户留存的地基能力。  
- **E5 视听与播放器体系（有声圣经/资源音视频）**：按通勤/睡前/低注意力场景的“媒体级”体验基线。  
- **E6 笔记体系（记录沉淀）**：让用户“写得下、找得到、带得走”，形成长期属灵资产。  
- **E7 高亮标注（标注/颜色/跨译本）**：让用户精确标注并可跨译本使用，支撑深度研经与复习。  
- **E8 资源体验（发现/搜索/分类/关联/收藏/分享）**：让已上线内容可被找到、被用起来、嵌入灵修流程。  
- **E9 Web/PWA/平板与跨端连续性**：Web不再是阉割版；PWA用于政策韧性与获客；平板用于深度研经与教会场景（投屏/备课）。  
- **E10 付费/奉献闭环（圣经App范围）**：把“想奉献/想购买（如大辞典）”的意愿变成可完成的路径，并降低支付失败与客诉。  
- **E11 体验可观测（客户端关键链路SLO/看板）**：把搜索/同步/播放等关键链路状态量化可见，支持快速定位与工程化防复发。  
- **E12 用户行为观察与实验（归因/漏斗/AB 测试）**：用可用的归因、漏斗与实验框架验证留存/转化假设，让增长投入可衡量、可复盘。  
- **E13 用户反馈闭环（反馈处理/自助排障）**：把反馈“收得住、分得清、回得去”，并用FAQ/错误码/诊断引导降低重复工单与用户挫败。  
- **E14 国际化与本地化体验（Ilumine/繁体/多语言/RTL）**：把“语言可用”升级为“处境可用”，形成可复制的国际化交付能力。  
- **E15 研读与理解圣经（注释/交叉引用/原文工具）**：为深度研经提供可信赖的理解工具与高效率工作台。

### **E1 可达性与稳定性底座（网络/性能）**

- **E1-S01｜国内网络“资源不可达”自助提示与合规指引（不提供离线下载）**  
  - **任务说明**：当检测到资源加载异常（疑似 GFW/弱网）时，提供明确的原因解释与可执行的自助排障步骤（重试/切换网络/切换线路/使用可访问的 Web 镜像或 PWA）。对涉及版权的资源给出合规说明：不提供离线下载/导出通道。  
  - **涉及平台**：App | Web  
  - **为什么**：系统性网络不可达会让多个功能同时失败；在“不可达”场景下给出可解释、可执行的下一步，能显著降低挫败感与重复工单，同时避免因离线下载触碰版权风险。  
  - **优先级**：P0  
  - **优先级说明**：该项属于核心链路阻塞相关的高优先级问题，直接影响核心指标或用户能否完成关键任务，需要优先止血并纳入门禁验收。
